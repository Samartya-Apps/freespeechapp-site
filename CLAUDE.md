## Project Overview

Marketing website for **Avaz FreeSpeech** by Samartya — an iPad/iPhone app helping children with language disabilities learn English grammar through interactive picture tiles.

**Stack:** Astro 5 + Tailwind CSS v4 + Cloudflare Pages (static output)

## Design Assets

- `one-pager/Scroll 0.jpg` through `Scroll 6.jpg` — vertical scroll design mockups (reference only, not served)

## Architecture

```
src/
  content.config.ts         — Astro Content Collections schema (blog)
  content/blog/             — Markdown blog posts
  data/faq.json             — FAQ items (plain JSON, imported directly)
  layouts/BaseLayout.astro  — HTML shell + Google Fonts + global CSS
  components/
    Nav.astro               — Sticky navigation
    Footer.astro
    AppStoreButton.astro    — Fixed bottom-right App Store CTA
    FeatureTabs.astro       — Accessible tab switcher (vanilla JS)
  pages/
    index.astro             — Home (7 sections)
    common-core.astro
    faq.astro
    blog/index.astro
    blog/[...slug].astro
  styles/global.css         — Tailwind v4 @theme tokens + brand styles
public/images/              — Placeholder SVGs for all sections
```

## Dev Commands

```bash
npm run dev       # local dev server → http://localhost:4321
npm run build     # production build → dist/
npm run preview   # preview prod build locally
npx wrangler pages deploy dist   # deploy to Cloudflare Pages
```

## Key Notes

- Tailwind **v4** is used — no `tailwind.config.mjs`. All brand tokens are in `src/styles/global.css` under `@theme { }`.
- Content Collections config is at `src/content.config.ts` (Astro 5 convention, NOT inside `src/content/`).
- FAQ JSON lives at `src/data/faq.json` (outside `src/content/` to avoid auto-collection detection).
- `astro.config.mjs` sets `output: 'static'` — all pages are pre-rendered.
- The `wrangler.jsonc` (not `.toml`) was generated by `@astrojs/cloudflare`.
- Node.js is managed via nvm; activate with `source "$HOME/.nvm/nvm.sh"` if `node` is not found.

## Accessibility Checklist

- Semantic HTML: `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>` used throughout
- All `<img>` tags have descriptive `alt` text
- Nav has `aria-label="Main navigation"`; active link gets `aria-current="page"`
- FeatureTabs uses `role="tablist"` / `role="tab"` / `role="tabpanel"` + arrow-key navigation
- FAQ uses native `<details>`/`<summary>` — no JS required
- `@media (prefers-reduced-motion: reduce)` guard in global.css
- YouTube `<iframe>` has `title` attribute and no autoplay
